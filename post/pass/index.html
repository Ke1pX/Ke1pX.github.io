使用 pass 密码管理器

2022 年 12 月 18 日 • 预计阅读时间 4 分钟

pass 是一款命令行密码管理器，pass 中的密码都保存在单独的 GnuPG 加密文件中。这些加密文件以树状结构展示，可以从一台计算机复制到另一台计算机，并通过命令行程序管理密码。

pass 中所有密码都位于 ~/.password-store 目录下，可以使用 git 将该目录设为 Git Repository 以自动跟踪密码操作。

主页：https://www.passwordstore.org/

开源代码：https://git.zx2c4.com/password-store

安装 pass

Arch Linux 环境下只需要输入 sudo pacman -S pass 即可，同时会安装依赖包 gnupg、git、tree 。其他环境的安裝方式可以参考官方说明。

1

sudo pacman -S pass

生成 GnuPG 密钥

在初始化 pass 之前，需要配置好 GnuPG 密钥，且密钥信任程度（trust level）必须为 ultimate。

参考 - ArchWiki - Create a key pair

1

gpg --full-gen-key

初始化 pass

1

pass init <gpg-id or email>

使用 pass

添加密码

使用 pass insert 命令添加一个邮箱密码，命令执行后要求输入两次邮箱密码接着会以指定的文件层级创建目录存放加密后的密码文件。

1

pass insert Email/abc-proton.me

1
2
3
4

pass ls
Password Store
└── Email
    └── abc-proton.me

查看密码

使用 pass Email/abc-proton.me 或 pass show Email/abc-proton.me 命令查看密码之前添加的密码，查看密码会要求输入 GnuPG 密钥的密码进行身份验证。

1
2
3
4

pass Email/abc-proton.me
abc
pass -c Email/abc-proton.me
Copied Email/abc-proton.me to clipboard. Will clear in 45 seconds.

查看密码操作会默认将密码打印到终端，可以使用 -c 参数将密码复制到剪贴板，密码会在 45 秒后从剪贴板清除。

生成密码

例如，使用 pass generate Servers/tokyo01-root 命令生成一个 16 位的密码。

1
2
3
4

pass generate Servers/tokyo01-root 16
mkdir: created directory '/home/$USERNAME/.password-store/Servers'
The generated password for Servers/tokyo01-root is:
O6BZv&W+:^F#h7IC

It’s possible to generate passwords with no symbols using --no-symbols or -n, and we can copy it to the clipboard instead of displaying it at the console using --clip or -c.

- passwordstore.org

其它操作

使用 pass edit、pass rm 、pass cp 、pass mv、pass find 等命令分别对密码文件进行编辑、删除、复制、剪切、查找操作，可使用命令 man pass 查看帮助信息。

初始化 Git 仓库

pass 支持使用 Git 对密码进行版本控制，为了方便我的手机及其它主机同步密码，我还需要配置一个私人 Git 服务器。

使用命令对 pass 主目录 ~/.password-store 执行初始化操作。

1

pass git init

配置 Git 服务器

Arch Linux 环境下配置 Git 服务器，需要将 git 用户过期日期设置为永不过期，重新启用 git 用户，再为 git 用户设置密码。

1
2

sudo chage -E -1 git
sudo passwd git

1
2

grep git /etc/passwd
git:x:974:974:git daemon user:/srv/git:/usr/bin/git-shell

Arch Linux 环境下 git 用户的登陆 shell 被设置为了 git-shell，这是一个受限制的 shell 且无法进行 SSH 登陆，详情参考：Git on the Server - Setting Up the Server

为了使 SSH 能够读取到文件 .ssh/authorized_keys ，需要将 git 用户的主目录变更为 /srv/git，并手动创建文件 .ssh/authorized_keys 将客户端 SSH 公钥写入 git 用户的 .ssh/authorized_keys 文件中，以确保客户端可以执行推送和拉取操作。

1
2
3
4
5
6
7

sudo mkdir -p /srv/git
sudo usermod git -d /srv/git
cd /srv/git
sudo mkdir .ssh && sudo chmod 700 .ssh
sudo touch .ssh/authorized_keys && sudo chmod 600 .ssh/authorized_keys
sudo bash -c "cat /home/$USERNAME/.ssh/id_rsa.pub >> /srv/git/.ssh/authorized_keys"
sudo bash -c "cat /data/Share/key/phonessh.asc >> /srv/git/.ssh/authorized_keys"

初始化一个 bare 的 Git 远程仓库，并配置目录权限。

1
2

sudo git init --bare passwordstore.git
sudo chown -R git:git /srv/git

为 pass 添加远程仓库（passwordstore.git），最后将 pass 的密码文件推送至远程仓库，且确保 SSH 服务已正常启动即可。

1
2
3

pass git remote add origin git@localhost:/srv/git/passwordstore.git
pass git push -u origin master
systemctl status sshd

至此，Git 服务器的基础配置就完成了，后续其它客户端只需要上传 SSH 公钥到 Git 服务器就可以通过这个仓库同步密码啦～

手机端密码同步

安卓手机使用 Password Store 作为 pass 客户端，另外还需要使用 OpenKeychain 以生成和管理 GnuPG 密钥，更直观的操作可以参考 YouTube 视频：Self Host A Minimalist, Password Manager: Git + Command Line!

IOS 则使用 Pass for iOS 作为客户端，客户端需要生成一对密钥以加解密密码文件，我使用安卓手机上已安装的 OpenKeychain 为 IPhone 生成 PGP 密钥，接着将密钥备份到 pass 主机，再使用备份密码解密密钥备份文件获取密钥信息，参考 - Export Android Openkeychain/PGP Private Key to PC using GunPG (Linux)。

然后将 IPhone 的公钥导入 pass 主机，编辑信任度为 绝对信任 重新初始化 pass，最后推送到 Git 服务器， IPhone 上配置 Pass for iOS 导入公钥和私钥后拉取密码文件即可。

1
2
3
4
5

gpg --decrypt /data/Share/key/backup_2022-12-18.sec.pgp
gpg --import /data/Share/key/iphoneSE.asc
gpg --edit-key "IPhone key"
pass init "PC Pass Key" "Android pass key" "IPhone key"
pass git push

此外，如果需要从手机端生成或添加密码并推送到服务器，需要将 pass 主机的 GnuPG 私钥导出并添加到 OpenKeychain 中。

1

gpg --export-secret-keys -a -o private.key

插件

pass 的拓展功能可以通过安装插件的方式实现，插件列表请参见 Extensions for pass 及 Awesome password-store。

例如，使用插件 browserpass 即可实现在浏览器上读取 pass 密码文件并填入登陆信息。 在 Arch Linux 环境下安装 browserpass：

1

yay -S browserpass browserpass-firefox

最后安装浏览器插件 Browserpass 即可。
