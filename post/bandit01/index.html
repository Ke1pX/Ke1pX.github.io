Wargame-Bandit Lv 01-10

2020 年 08 月 14 日 • 预计阅读时间 4 分钟

Wargame 以一种充满乐趣的游戏形式帮助用户通过实践学习安全知识，其中 Bandit 是一个适合初学者练习的 Linux 基础知识的游戏(目前设有 34 个关卡)，游戏的每一个关卡有相应的游戏提示。

游戏地址：https://overthewire.org/wargames/bandit/

任务：找到通往下一关卡的密码

Level 0

The goal of this level is for you to log into the game using SSH. The host to which you need to connect is bandit.labs.overthewire.org, on port 2220. The username is bandit0 and the password is bandit0. Once logged in, go to the Level 1 page to find out how to beat Level 1.

—Level 0

第一关，要求使用 SSH 连接到靶机并给出如下信息：

1
2
3
4
5
6
7

Host：bandit.labs.overthewire.org

Port: 2220

Username: bandit0

Password: bandit0

SSH 相关内容：《SSH 入门教程》发布了

使用终端登录：

1

ssh bandit0@landit.labs.overthewire.org

Level 0 - 1

The password for the next level is stored in a file called readme located in the home directory. Use this password to log into bandit1 using SSH. Whenever you find a password for a level, use SSH (on port 2220) to log into that level and continue the game.

—Level 0 - 1

提示下一关密码在 readme 文件中，使用 ls 命令列出当前目录下的文件，使用 cat 命令查看文件内容获取下一关密码并使用 SSH 连接到下一关卡

1
2
3

ls 
cat readme
ssh bandit1@localhost

Level 1 - 2

The password for the next level is stored in a file called - located in the home directory

—Level 1 - 2

提示密码在一个名为 “-” 的文件中，发现这是一个连字符当输入 cat - 运行系统将无法理解这个文件名，为此在命令中添加文件路径使用命令 cat ./- 表示查看当前目录下的 - 文件，获取到下一关的密码并使用 SSH 连接到下一关卡

1
2
3

ls 
cat ./-
ssh bandit2@localhost

Level 2 - 3

The password for the next level is stored in a file called spaces in this filename located in the home directory

—Level 2 - 3

提示密码在一个名为 “spaces” 的文件中，ls 查看发现文件名含有空格字符，可使用 “\” 将文件名中的空格字符进行转义或者输入文件名前面字段后使用 TAB 键补全文件名，另外也可以将文件名使用引号引起来，获取到下一关密码并使用 SSH 连接到下一关卡

1
2
3
4

ls
cat spaces\ in\ this\ filename
cat "spaces in this filename"
ssh bandit3@localhost

Level 3 - 4

The password for the next level is stored in a hidden file in the inhere directory.

—Level 3 - 4

提示密码在 inhere 目录下一个隐藏文件，使用 ls 命令 -a 选项显示 inhere 目录下的隐藏文件并使用 cat 命令查看隐藏文件内容获取下一关密码使用 SSH 连接到下一关卡

1
2
3
4

ls
ls -a inhere
cat inhere/.hidden
ssh bandit4@localhost

Level 4 - 5

The password for the next level is stored in the only human-readable file in the inhere directory. Tip: if your terminal is messed up, try the “reset” command.

—Level 4 - 5

提示密码在 inhere 目录下的一个人类可读的文件，ls 查看一下 inhere 目录发现很多文件，使用 file 命令查看这些文件的文件类型 find inhere/* 发现仅有一个文件是 ASCII 文本文件查看该文件获取密码

参考：https://phoenixnap.com/kb/linux-file-command

1
2
3
4
5

ls
ls inhere/
file inhere/*
cat inhere/-file07
ssh bandit5@localhost

Level 5 - 6

The password for the next level is stored in a file somewhere under the inhere directory and has all of the following properties:

human-readable

1033 bytes in size

not executable

—Level 5 - 6

根据提示密码在 inhere 目录下的某个文件中，且具有以下属性：

  • 人类可读
  • 文件字节大小为 1033 字节
  • 不可执行

根据文本文件大小使用 find 命令查询使用 -size 选项 c 表示匹配文件大小单元为字节,找到下一关密码使用 SSH 连接到下一关卡

参考：find(1) — Linux manual page

1
2
3
4
5

ls
ls inhere
file inhere/*
cat inhere/maybehere07/.file2
ssh bandit6@localhost

Level 6 - 7

The password for the next level is stored somewhere on the server and has all of the following properties:

owned by user bandit7

owned by group bandit6

33 bytes in size

—Level 6 - 7

提示密码在服务器的某个位置，且具有以下属性：

  • 所属用户为 bandit7
  • 所属组为 bandit6
  • 文件字节大小为 33 个字节

同样使用 find 命令查询整个系统，-user 表示匹配所属用户、-group 表示匹配所属组，获取密码后使用 SSH 连接到下一关卡

1
2

find / -user bandit7 -group bandit6 -size 33c
ssh bandit7@localhost

Level 7 - 8

The password for the next level is stored in the file data.txt next to the word millionth

—Level 7 - 8

根据提示密码在当前目录下的 data.txt 文件中单词 “millionth” 的旁边，使用管道符连接命令将 cat 的输出通过 grep 执行关键词搜索过滤掉多余内容，获取密码后使用 SSH 连接到下一关卡

1
2

cat data.txt | grep millionth
ssh bandit8@localhost

Level 8 - 9

The password for the next level is stored in the file data.txt and is the only line of text that occurs only once

—Level 8 - 9

密码在 data.txt 文件中且仅出现一次，先使用 sort 命令对文本内容进行排序再使用 uniq 过滤不重复的语句 -u 选项表示仅显示出现一次的行列，获取密码后使用 SSH 连接到下一关卡

1
2

cat data.txt | sort | uniq -u
ssh bandit9@localhost

Level 9 - 10

The password for the next level is stored in the file data.txt in one of the few human-readable strings, preceded by several ‘=’ characters.

—Level 9 - 10

提示密码在 data.txt 文件中，但少数几人能读懂(非文本文件)密码前面有 “=” 字符，关键词 strings，使用 strings 命令打印文件中可打印的字符串,获取密码后使用 SSH 连接到下一关卡

参考：https://www.javatpoint.com/linux-strings-command

1
2

strings data.txt | grep =
ssh bandit10@localhost
